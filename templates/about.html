About the Project

A comprehensive overview of our LLM-Based Multi-Database Chatbot System

System Overview

This is an advanced conversational AI system that combines multiple data sources and intelligent routing to provide comprehensive answers to healthcare-related queries. The system seamlessly integrates SQL database queries with PDF document retrieval, using a sophisticated routing mechanism to determine the most appropriate data source for each question.

System Architecture

User Interface

Modern web interface built with HTML5, CSS3, and JavaScript. Features real-time chat capabilities with typing indicators and responsive design.

Backend API

FastAPI-based REST API that handles chat requests, manages agent initialization, and coordinates between different data sources.

Router Agent

Intelligent agent that analyzes user queries and routes them to the appropriate tool (SQL Agent or PDF RAG) based on question type.

SQL Agent

LangChain SQL agent that converts natural language to SQL queries and retrieves structured data from the healthcare database.

PDF RAG

Retrieval Augmented Generation system using FAISS vector store and HuggingFace embeddings for document search.

Models & Technologies Used

Language Model

OpenAI GPT-3.5-turbo

  • Used for natural language understanding and generation
  • Powers the router agent for intelligent query classification
  • Generates human-like responses from structured data

Embeddings Model

sentence-transformers/all-MiniLM-L6-v2

  • Creates vector embeddings for PDF document search
  • Optimized for semantic similarity and retrieval
  • Enables efficient document querying

Vector Database

FAISS (Facebook AI Similarity Search)

  • High-performance similarity search for document retrieval
  • Stores and indexes PDF document embeddings
  • Enables fast semantic search across documents

System Logic & Workflow

1

User Query Reception

User submits a natural language question through the web interface. The query is sent to the FastAPI backend.

2

Router Agent Analysis

The router agent analyzes the query to determine whether it's a database question (patient data, medications, visits) or a document question (policies, rights, coverage).

3

Tool Selection & Execution

Based on the analysis, the query is routed to either the SQL Agent (for database queries) or the PDF RAG system (for document queries).

4

Response Generation

The selected tool processes the query and generates a comprehensive response with relevant details and source information.

5

Response Delivery

The response is formatted and sent back to the frontend, where it's displayed in the chat interface with tool details.

Why This Approach?

Intelligent Routing

Instead of a single monolithic system, we use specialized agents for different data types. This ensures optimal performance and accuracy for each query type.

Data Security

By separating database queries from document searches, we maintain better control over data access and can implement appropriate security measures for each data source.

Scalability

The modular architecture allows easy addition of new data sources and tools without affecting existing functionality.

Performance

Specialized tools are more efficient than a general-purpose system. SQL queries are faster for structured data, while RAG is better for document search.
